## 分支策略

团队协作中，清晰的分支策略很重要：

- `main` — 生产环境代码
- `develop` — 开发分支
- `feature/*` — 功能分支
- `fix/*` — 修复分支

## 常用命令

### 交互式 rebase

整理提交历史：

```bash
git rebase -i HEAD~3
```

### 暂存工作区

临时切换分支时：

```bash
git stash
git checkout other-branch
# 做完事情后
git checkout -
git stash pop
```

### 查找引入 bug 的提交

```bash
git bisect start
git bisect bad          # 当前版本有 bug
git bisect good v1.0.0  # 这个版本没有 bug
# Git 会自动二分查找
```

## Commit Message 规范

推荐使用 Conventional Commits：

| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | 修复 bug |
| docs | 文档更新 |
| refactor | 重构 |
| chore | 构建/工具变更 |

好的 commit message 能让代码审查更高效。
